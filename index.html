⚙️ GitHub Pages Deployment — How It Works
01
Create Repo
Name it username.github.io or any repo with Pages enabled
02
Choose Generator
Jekyll (native support) or any SSG that outputs to /docs or gh-pages branch
03
Write Posts
Markdown files in /_posts/ — push to main to auto-publish
04
Custom Domain
Add CNAME file + DNS record to point sudosessions.dev → GitHub Pages

// Setup Commands
# 1. Install Jekyll $ gem install jekyll bundler # 2. Create site $ jekyll new sudosessions $ cd sudosessions # 3. Init git and push to GitHub $ git init && git remote add origin git@github.com:username/sudosessions.git $ git add . && git commit -m "feat: initial sudosessions setup" $ git push -u origin main # 4. Add custom domain (optional) $ echo "sudosessions.dev" > CNAME $ git add CNAME && git commit -m "feat: add custom domain" && git push
